## 一、项目概览

Idel-DreamMaker = 藏在系统托盘里的挂机叙事陪伴应用。**大厅 (Hub) + 副本内 (Scenario)** 双层架构：在大厅管理副本，进入副本后零交互挂机升级、随机弹出故事事件。

原名 IdleWorker，v0.3.0 起改名 Idel-DreamMaker。

## 四、体系架构

### 双层结构

```
┌─────────────────────────────────────────────┐
│              大厅 (Hub)                      │
│  ┌───────────────────────────────────────┐  │
│  │  全局玩家名（唯一）                    │  │
│  │  副本列表 ─ 选取 / 抽取 / 导入 / AI   │  │
│  │  大厅等级（跨副本累加，永不重置）      │  │
│  │  大厅称号（各副本称号聚合视图）        │  │
│  │  语言设置                             │  │
│  └───────────────────────────────────────┘  │
│                     │                        │
│        选择副本 → 进入（可选别名）           │
│                     ↓                        │
│  ┌───────────────────────────────────────┐  │
│  │              副本内 (Scenario)          │  │
│  │  副本内别名（独立于全局名）             │  │
│  │  EXP 1/s → 等级/称号/事件/成就         │  │
│  │  退出 → 等级重置，EXP 累入大厅         │  │
│  └───────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

### 核心数据流

```
启动 → init() → 判断 is_in_hub
                  ↓
       Hub = true → 显示大厅界面
       Hub = false → 显示副本界面 + 游戏循环开始
                       ↓
              游戏循环 (JS setInterval, 500ms tick)
                       ↓
           exp += 1/s ─┬→ 等级检测 → emit(level-up)
                        ├→ 每 1s → emit(game-tick) → 前端 updateUI()
                        ├→ 每 60s → 概率触发事件 → emit(event-triggered)
                        ├→ 每 tick → 成就检测 → emit(achievement-unlocked)
                        └→ 每 30s → save_game()

      退出副本时：
        hub.total_exp += scenario.exp_earned
        scenario.exp_earned = 0
        scenario.level = 1
        is_in_hub = true → 切换显示大厅界面
```

## 五、关键模块详解

### 5.1 大厅 (Hub)

| 功能 | 实现位置 | 说明 |
|------|---------|------|
| 副本列表 | `main.js:renderScenarioPanel()` | 卡片列表，显示各副本名称/描述/进度 |
| 副本选择 | `lib.rs:select_scenario` | 进入副本，可选别名，重置副本内状态 |
| 副本抽取 | `lib.rs:draw_scenario` | 随机选一个副本（免费不限次） |
| 大厅等级 | `game.rs:GameState.hub` | `hub_total_exp` 跨副本累加，公式 `sqrt(exp/100)+1` |
| 大厅称号 | `main.js:renderHubTitles()` | 各副本解锁称号聚合视图，按副本分组展开/折叠 |
| 副本数据 | `build.rs` + `scenario.rs` | 构建时 .md → bincode 序列化，运行时反序列化 |

**大厅内游戏规则：**
- `is_in_hub = true` → 游戏循环跳过 EXP 增长、事件触发、成就检测
- 仅更新 UI（计时器继续走）
- 所有按钮可用：设置、副本选择、称号、隐藏

### 5.2 副本内 (Scenario)

| 模块 | 实现位置 | 说明 |
|------|---------|------|
| GameState | `game.rs:GameState` | 副本内状态：level/exp/称号/事件/成就 |
| 游戏循环 | `game.rs:start_game_loop()` | 500ms tick，仅当 `!is_in_hub` 时执行 |
| 事件触发 | `game.rs:check_and_trigger_event()` | 每 60s 概率触发 |
| 成就检测 | `game.rs:check_achievements()` | 每 tick 检测 |
| 存档读写 | `game.rs:load_save()/save_game()` | 单文件 JSON |

**退出副本逻辑：**
```rust
fn exit_scenario(game: &mut GameState) {
    game.hub_total_exp += game.total_exp_earned;
    game.scenario_id = "hub".to_string();
    game.level = 1;
    game.exp = 0.0;
    game.total_exp_earned = 0.0;
    game.total_runtime_ms = 0;
    game.triggered_events.clear();
    game.unlocked_achievements.clear();
    game.is_in_hub = true;
}
```

### 5.3 语言系统

| 配置项 | 存储位置 | 说明 |
|--------|---------|------|
| `language` | `GameState.language` | UI 语言，默认 `zh`，可选 `en` |

前端通过 `const LANG = { zh: {...}, en: {...} }` 映射表实现文本切换。

### 5.4 窗口行为

- 默认尺寸：320×840
- 最小尺寸：280×400
- 自定义标题栏（`decorations: false`），仅留 [×] 关闭按钮
- Mini Bar：250×80 半透明置顶横条，可拖拽
- 贴边自动隐藏：已移除（v0.3.2）

## 十、注意点

- **本项目为 vibe coding 项目。** 创作者不具备编程技术背景。AI 在生成代码/文档/内容时需遵循：
  - 输出自解释、可维护，避免不必要的复杂度
  - 涉及"作者操作"的流程需给出清晰步骤和预期结果
  - 优先选择实现简单、出错后容易修复的方案
- 交易引擎在 Electron 主进程运行，通过 IPC 与渲染进程通信
- 副本 `.md` 文件为作者内部制作工具，构建时解析为 JSON
- 游戏是零交互的——进入副本后没有按钮/点击，纯挂机
- Electron 打包安装包约 150MB（含 Chromium）
- Maple Mono NF CN 字体文件存放于 `public/fonts/`，打包时包含

## 一、项目概览

Idel-DreamMaker = 藏在系统托盘里的挂机叙事陪伴应用。**大厅 (Hub) + 剧本内 (Scenario)** 双层架构：在大厅管理剧本，进入剧本后零交互挂机升级、随机弹出故事事件。

原名 IdleWorker，v0.3.0 起改名 Idel-DreamMaker。

## 四、体系架构

### 双层结构

```
┌─────────────────────────────────────────────┐
│              大厅 (Hub)                      │
│  ┌───────────────────────────────────────┐  │
│  │  全局玩家名（唯一）                    │  │
│  │  剧本列表 ─ 选取 / 抽取 / 导入 / AI   │  │
│  │  大厅等级（跨剧本累加，永不重置）      │  │
│  │  大厅称号（各剧本称号聚合视图）        │  │
│  │  语言设置 / AI 设置                   │  │
│  └───────────────────────────────────────┘  │
│                     │                        │
│        选择剧本 → 进入（可选别名）           │
│                     ↓                        │
│  ┌───────────────────────────────────────┐  │
│  │              剧本内 (Scenario)          │  │
│  │  剧本内别名（独立于全局名）             │  │
│  │  EXP 1/s → 等级/称号/事件/成就         │  │
│  │  退出 → 等级重置，EXP 累入大厅         │  │
│  └───────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

### 核心数据流

```
启动 → init() → 判断 is_in_hub
                  ↓
       Hub = true → 显示大厅界面
       Hub = false → 显示剧本界面 + 游戏循环开始
                       ↓
              游戏循环 (Rust 线程, 500ms tick)
                       ↓
           exp += 1/s ─┬→ 等级检测 → emit(level-up)
                        ├→ 每 1s → emit(game-tick) → 前端 updateUI()
                        ├→ 每 60s → 概率触发事件 → emit(event-triggered)
                        ├→ 每 tick → 成就检测 → emit(achievement-unlocked)
                        └→ 每 30s → save_game()

      退出剧本时：
        hub.total_exp += scenario.exp_earned
        scenario.exp_earned = 0
        scenario.level = 1
        is_in_hub = true → 切换显示大厅界面
```

## 五、关键模块详解

### 5.1 大厅 (Hub)

| 功能 | 实现位置 | 说明 |
|------|---------|------|
| 剧本列表 | `main.js:renderScenarioPanel()` | 卡片列表，显示各剧本名称/描述/进度 |
| 剧本选择 | `lib.rs:select_scenario` | 进入剧本，可选别名，重置剧本内状态 |
| 剧本抽取 | `lib.rs:draw_scenario` | 随机从未拥有的剧本中选一个（免费不限次） |
| 大厅等级 | `game.rs:GameState.hub` | `hub_total_exp` 跨剧本累加，公式 `sqrt(exp/100)+1` |
| 大厅称号 | `main.js:renderHubTitles()` | 各剧本解锁称号的聚合视图，按剧本分组展开/折叠 |

**大厅内游戏规则：**
- `is_in_hub = true` → 游戏循环跳过 EXP 增长、事件触发、成就检测
- 仅更新 UI（计时器继续走）
- 所有按钮可用：设置、剧本选择、称号、状态、隐藏

### 5.2 剧本内 (Scenario)

| 模块 | 实现位置 | 说明 |
|------|---------|------|
| GameState | `game.rs:GameState` | 剧本内状态：level/exp/称号/事件/成就 |
| 游戏循环 | `game.rs:start_game_loop()` | 500ms tick，仅当 `!is_in_hub` 时执行 |
| 事件触发 | `game.rs:check_and_trigger_event()` | 每 60s 概率触发 |
| 成就检测 | `game.rs:check_achievements()` | 每 tick 检测 |
| 存档读写 | `game.rs:load_save()/save_game()` | 单文件 JSON |

**退出剧本逻辑：**
```rust
fn exit_scenario(game: &mut GameState) {
    game.hub_total_exp += game.total_exp_earned;
    game.scenario_id = "hub".to_string();
    game.level = 1;
    game.exp = 0.0;
    game.total_exp_earned = 0.0;
    game.total_runtime_ms = 0;
    game.triggered_events.clear();
    game.unlocked_achievements.clear();
    game.is_in_hub = true;
}
```

### 5.3 语言系统

| 配置项 | 存储位置 | 说明 |
|--------|---------|------|
| `language` | `GameState.language` | UI 语言，默认 `zh`，可选 `en` |
| `ai_output_language` | `GameState.ai_output_language` | AI 输出语言，默认 `zh` |

前端通过 `const LANG = { zh: {...}, en: {...} }` 映射表实现文本切换。

### 5.4 窗口与贴边自动隐藏

- 默认尺寸：320×840
- 最小尺寸：280×400
- 贴边检测：鼠标距屏幕边缘 < 5px 持续 500ms → 窗口缩为 4px 细边
- 展开检测：鼠标移回细边区域 → 窗口恢复原大小
- 实现：前端 `mousemove` 监听 + `window.setSize()` / `window.show()` / `window.hide()`

## 十、注意点

- `rand::gen()` 在 Rust 2024 edition 中为保留关键字，需使用 `r#gen()`
- 剧本 `.md` 文件需严格遵循 `剧本MD格式模板.md` 规范
- 游戏是零交互的——进入剧本后没有按钮/点击，纯挂机
- 所有事件文本用 AI 批量生成后放入剧本文件，不硬编码
- Steamworks 集成为可选项，条件编译 `#[cfg(feature = "steam")]`
- Maple Mono NF CN 字体文件存放于 `src-tauri/fonts/`，打包时包含

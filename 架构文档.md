> **版本：** 0.1.0-beta
> **生成日期：** 2026-06-11

## 一、项目概览

IdleWorker = 藏在系统托盘/菜单栏里的纯文字挂机叙事游戏。零交互，时间累积等级，随机弹出故事事件，支持多剧本和全球节假日系统。

## 四、工作流

### 核心数据流

```
启动 → init() → invoke(get_full_state) → 初始化状态 & 注册事件监听
                 ↓
        游戏循环 (Rust 线程, 500ms tick)
                 ↓
     exp += 1/s ─┬→ 等级检测 → emit(level-up)
                  ├→ 每 1s → emit(game-tick) → 前端 updateUI()
                  ├→ 每 60s → 概率触发事件 → emit(event-triggered) → 弹窗 + 日志
                  ├→ 每 tick → 成就检测 → emit(achievement-unlocked)
                  └→ 每 30s → save_game() 写入 %APPDATA%/IdleWorker/save.json
```

### 开发命令

| 场景 | 命令 | 说明 |
|------|------|------|
| 启动开发 | `npm run tauri dev` | 同时启动 Vite 热更新 + Tauri 应用 |
| 构建产物 | `npm run tauri build` | 生成安装包到 `src-tauri/target/release/` |
| 仅前端 | `npm run dev` | Vite 开发服务器 (localhost:1420) |
| 仅 Rust | `cargo build`（在 src-tauri 目录下） | 编译 Rust 后端 |

## 五、关键模块详解

### 5.1 `src-tauri/src/game.rs` — 游戏引擎核心

**GameState 结构体：**
- `player_name: String` — 玩家自定义名称
- `scenario_id: String` — 当前剧本 ID
- `level: u64` — 当前等级
- `exp: f64` — 当前经验值
- `total_exp_earned: f64` — 累计总经验
- `total_runtime_ms: u64` — 累计运行毫秒
- `triggered_events: Vec<String>` — 已触发的 once 事件 ID
- `unlocked_achievements: Vec<String>` — 已解锁成就 ID
- `selected_font_theme: String` — 字体主题

**AppState：**
```
game: Mutex<GameState>          // 可变的游戏状态
scenario: Mutex<Scenario>       // 当前剧本
all_scenarios: Vec<Scenario>    // 全部剧本（只读，启动时加载）
```

**游戏循环参数：**
- 循环间隔 500ms
- EXP 速率 1/s
- 等级公式：`floor(sqrt(total_exp / 100)) + 1`
- 事件触发概率递减：前 12h 40% → 12-72h 30% → 72h+ 15%
- 事件选择按 weight 权重随机
- 自动存档每 30s

### 5.2 `src-tauri/src/scenario.rs` — 剧本数据模型

- `Scenario` — 剧本完整定义（id, name, name_cn, description, titles, events, achievements）
- `TitleDef` — 称号定义（level, name, color, desc）
- `EventDef` — 事件定义（id, min_level, min_hours, weight, once, text）
- `AchievementDef` — 成就定义，含 tagged enum `AchievementCondition`
- 剧本 JSON 通过 `include_str!` 编译时嵌入

### 5.3 `src-tauri/src/lib.rs` — 命令与托盘

**9 个 Tauri 命令：**
- `get_full_state` — 初始化全量状态
- `get_scenario_list` — 剧本列表
- `get_scenario_detail` — 剧本详情（含称号/成就完整数据）
- `set_player_name` / `set_font_theme` — 玩家设置
- `select_scenario` — 切换剧本（重置等级）
- `show_window` / `hide_window` — 窗口控制

**系统托盘：**
- 左键单击：切换窗口显示/隐藏
- 右键菜单：Show/Hide | ─── | Quit
- 窗口关闭按钮：隐藏到托盘（不退出）

### 5.4 前端通信架构

- **invoke（前端 → 后端）**：一次性请求-响应，用于初始化、设置、面板数据等
- **emit/listen（后端 → 前端）**：持续推送，用于 game-tick（1s）、event-triggered（~60s）、level-up、achievement-unlocked

## 九、注意点

- `rand::gen()` 在 Rust 2024 edition 中为保留关键字，需使用 `r#gen()`
- 剧本 JSON 字段使用 camelCase，Rust 通过 `#[serde(rename)]` 映射
- 前端无框架依赖，纯原生 JS + CSS
- 游戏是零交互的——没有按钮点击玩法，纯挂机 + 偶尔阅读
- 所有事件文本用 AI 批量生成后放入剧本 JSON，不硬编码在代码里
- Steamworks 集成为可选项，使用条件编译 `#[cfg(feature = "steam")]`

# Idel-DreamMaker 节假日事件池

> 全球节假日 + 虚构游戏内节日的独立事件系统。不占用副本事件名额，开机时检测日期动态激活。

---

## 检测规则

| 项目 | 值 |
|------|-----|
| 检测时机 | 每次事件触发时 |
| 检测方式 | 匹配系统当前日期的 `月-日` |
| 匹配优先级 | 全球性 > 区域性 > 虚构游戏内节日 |
| 权重复盖 | 命中节假日时，该节日的 weight 大幅提升，优先于普通事件 |
| 同日多节日 | 取优先级最高的一个，不会重复弹 |

## 节假日事件 JSON 数据结构

```json
{
  "holidays": [
    {
      "id": "new_year",
      "name": "元旦",
      "date": { "month": 1, "day": 1 },
      "priority": "global",
      "weightMultiplier": 5,
      "icon": "🎆",
      "events": [
        "新年的第一缕阳光透过辐射云照亮了废土。",
        "你在日志上写下：这是新的一年。尽管世界没变，但至少你还在。"
      ],
      "perScenarioTexts": {
        "wasteland": [
          "今年的第一场辐射雨比往年来得更早一些。你在避难所门口贴了一张新的日历——虽然你也不确定今天是周几了。"
        ],
        "medieval": [
          "城堡的大厅里燃起了新的火炬。吟游诗人弹唱着新年的赞歌，你举起酒杯，向未知的一年致敬。"
        ],
        "cyberpunk": [
          "时代广场的巨型全息屏倒数着新年。人群欢呼，而你靠在巷子里的墙壁上，看着头顶绽放的数字烟花。"
        ],
        "cultivation": [
          "新年的第一缕灵气随着朝阳升起。你在洞府前打坐，感受着天地间微弱的脉动——新的轮回开始了。"
        ],
        "cthulhu": [
          "日历翻到了新的一页。但你真的相信人类的历法吗？那些古老的存在不会因为年份变了就放过你。"
        ]
      }
    }
  ]
}
```

## 副本专属化用规则

当 `perScenarioTexts` 中有对应副本的事件时，**优先使用定制版**。

定制版为空数组时，使用通用 `events`。

设计原则：节假日事件要贴合副本的世界观，但保留节日的核心情感（团圆、希望、纪念等）。

## 虚构游戏内节日

除了真实节日外，每个副本有 **2-3 个虚构专属节日**，只在对应副本激活时触发：

| 副本 | 节日名 | 日期 | 说明 |
|------|--------|------|------|
| Wasteland | 辐射尘静默日 | 随机 | 辐射水平异常降低，废土短暂平静 |
| Wasteland | 避难所开放日 | 随机 | 发现一个废弃的避难所 |
| Medieval | 骑士册封节 | 随机 | 王国举行大规模册封 |
| Medieval | 龙焰节 | 随机 | 纪念屠龙英雄 |
| Cyberpunk | 网络解放日 | 随机 | 全世界黑客的节日 |
| Cyberpunk | 义体大促销 | 随机 | 黑色幽默——企业打折日 |
| Cultivation | 飞升节 | 随机 | 纪念成功飞升的前辈 |
| Cultivation | 灵潮日 | 随机 | 灵气潮汐最强之日 |
| Cthulhu | 低语之夜 | 随机 | 古神低语最强之夜 |
| Cthulhu | 帷幕薄弱日 | 随机 | 现实与异界之间屏障最弱 |

虚构节日的日期为**运行时随机分配**（首次加载副本时确定，存入存档），而不是固定日期，增加意外感。

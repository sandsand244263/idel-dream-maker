# Idel-DreamMaker 副本 `.md` 格式模板

> 每个副本一个 `.md` 文件，frontmatter（YAML）定义元数据，正文 Markdown 标题分割各区块。
> AI 生成副本时，**必须严格遵循此模板**，否则格式验证不通过，不会生效。

## 字段说明

### Frontmatter（YAML）

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| `id` | 是 | string | 唯一标识，如 `wasteland`、`medieval`。只能含小写字母、数字、下划线 |
| `name` | 是 | string | 英文名，如 `Wasteland` |
| `name_cn` | 是 | string | 中文名，如 `废土` |
| `description` | 是 | string | 一段文字描述世界观 |
| `player_title` | 是 | string | 起始称号（Lv.1 默认称号），如 `拾荒者` |
| `language` | 是 | string | 副本语言，如 `zh`、`en` |

### Titles 表

| 列 | 必填 | 说明 |
|----|------|------|
| Level | 是 | 解锁所需等级 |
| Name | 是 | 称号名称 |
| Color | 是 | 显示用的十六进制颜色（含 `#`） |
| Description | 是 | 称号描述文字 |

**规则：** Level 列必须升序排列，最小为 1。至少需要 5 个称号。

### Events 表

| 列 | 必填 | 说明 |
|----|------|------|
| ID | 是 | 格式：`{副本id}_e{三位编号}`，如 `w_e001` |
| MinLevel | 否，默认 1 | 最低触发等级 |
| MinHours | 否，默认 0 | 最低挂机小时数 |
| Weight | 否，默认 5 | 权重（1-10），越大越容易触发 |
| Once | 否，默认 否 | 是否只触发一次（是/否） |
| Text | 是 | 事件文本，30-80 字 |

**规则：** 每个副本建议 500 条事件。`once=是` 的事件设计为关键剧情节点。

### Achievements 表

| 列 | 必填 | 说明 |
|----|------|------|
| ID | 是 | 格式：`{副本id}_a{三位编号}`，如 `wa_001` |
| Name | 是 | 成就名称 |
| Description | 是 | 成就描述 |
| Icon | 否，默认 ★ | 单字符图标 |
| ConditionType | 是 | 条件类型：`level` / `runtime` / `events` / `titles` |
| ConditionValue | 是 | 条件数值，与类型对应 |

**条件类型对照：**

| ConditionType | ConditionValue 含义 | 示例 |
|---------------|---------------------|------|
| `level` | 达到指定等级 | `10` → 达到 Lv.10 |
| `runtime` | 累计运行毫秒数 | `86400000` → 运行满 24 小时 |
| `events` | 触发不同事件的数量 | `10` → 触发过 10 个不同事件 |
| `titles` | 解锁称号的数量 | `5` → 获得过 5 个称号 |

## 常见错误检查

- [ ] Frontmatter 用 `---` 包裹，不是 `---` 拼写错误
- [ ] 表格分隔符行用 `|---` 格式
- [ ] 表格列数与表头列数一致
- [ ] Color 列含 `#` 前缀
- [ ] 所有 ID 唯一，不重复
- [ ] Title Level 升序排列
- [ ] Event Once 列只用 `是` / `否`
- [ ] Frontmatter `language` 字段值为 `zh` 或 `en`

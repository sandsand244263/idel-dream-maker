# Idel-DreamMaker 工作任务清单

> 跨版本工作追踪。新 AI 对话请先读此文件，找到当前未完成的任务。

---

> **重要背景：** 创作者本人不具备编程技术背景，本项目完全通过 vibe coding 方式制作。
> AI 在生成代码/内容/文档时需考虑这一前提——输出应自解释、可维护、避免不必要的复杂度。

## v0.3.7 — 副本引擎与废土填充

> **当前阶段：v0.3.7（全部完成 ✅）**
>
> 核心目标：构建时 .md 解析 + 副本内容保护 + 废土副本完整填充
>
> 注意：取消原计划的玩家侧 AI 生成/导入导出功能，副本改为作者闭源制作。
> 副本数据在构建时（build.rs）解析并序列化为二进制嵌入，运行时不可直接提取。

| Task | 内容 | 涉及文件 |
|------|------|---------|
| **1** | **Cargo 依赖 + build.rs 框架** | `Cargo.toml` (+bincode), `build.rs` (新建) | ✅ |
| **2** | **.md 解析器**：YAML frontmatter + 3 种 Markdown 表格解析 | `build.rs` (parse_*) | ✅ |
| **3** | **build-time 序列化**：.md → bincode → `scenarios_data.rs`（自动生成，OUT_DIR） | `build.rs` | ✅ |
| **4** | **运行时反序列化**：从生成字节加载 Scenario | `scenario.rs` | ✅ |
| **5** | **内容校验**：ID 校验、weight 范围、事件文本长度、成就条件合理性 | `build.rs` (validate_scenario) | ✅ |
| **6** | **核心引擎测试**（9 个）：等级计算、称号查找、事件过滤、成就检测 | `scenario.rs` (末尾 `#[test]`) | ✅ |
| **7** | **前端错误处理**：所有 `catch(e){}` 改为日志输出 | `main.js` | ✅ |
| **8** | **废土 500 事件**：wasteland.md 扩充到 500 条事件 | `scenarios/wasteland.md` | ✅ |

> 执行顺序：先 1-7（基础架构 + 测试 + 修复），再 8（内容填充）

## v0.4.0 — 像素宠物窗口（替换 Mini Bar）

> **当前阶段：v0.4.0（即将开始）**
>
> 核心目标：用像素宠物悬浮窗口取代现有的 Mini Bar，支持加载 Petdex 格式的社区宠物素材。
>
> 宠物来源：用户从 petdex.dev 下载，放入 `pets/{slug}/pet.json + spritesheet.webp`。
> 版权声明：UI 底部显示 "宠物来自 petdex.dev，版权归原作者所有"。

| Task | 内容 | 涉及文件 | 状态 |
|------|------|---------|------|
| 0 | **修复启动问题** | 替换新版 WebView2Loader.dll，兼容 v149 Edge 运行时 | `target/debug/` | ⬜ |
| 1 | **新建宠物窗口 HTML** | 极小尺寸窗口 (~240×180)，透明背景，置顶，鼠标穿透 | `src/pet.html` | ⬜ |
| 2 | **宠物渲染 + 动画逻辑** | CSS spritesheet 帧动画：idle 循环、点击跳跃、事件响应 | `src/pet.js` | ⬜ |
| 3 | **宠物窗口样式** | 透明背景、EXP 进度条、等级/称号显示、版权声明 | `src/pet.css` | ⬜ |
| 4 | **Tauri 后端命令** | 宠物窗口创建/显示/隐藏、状态同步事件 | `lib.rs`, `tauri.conf.json` | ⬜ |
| 5 | **Petdex 素材加载** | 读取 `pets/{slug}/pet.json` + spritesheet，解析帧信息 | `pet.js` | ⬜ |
| 6 | **"迷你"按钮改为唤出宠物** | 替换旧 Mini Bar 入口 | `main.js`, `index.html` | ⬜ |
| 7 | **兜底提示** | 无宠物时显示"请从 petdex.dev 下载宠物" | `pet.html` | ⬜ |
| 8 | **测试 + 清理旧 Mini Bar 代码** | 确认全部功能正常后删除旧代码 | | ⬜ |

## 验证标准

每个 Task 完成后：
```
cargo test          # Rust 测试全过
cargo check         # 编译通过
npx vite build      # 前端构建通过
git push origin main  # 推送到 GitHub
```

版本结束时额外：
```
cargo tauri build   # 打包验证
```

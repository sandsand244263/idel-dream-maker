### v0.4.0 — AI 接入 + 导入导出

> 规划中，未开始。默认 AI 方案为 DeepSeek（几乎免费，中文好），同时提供 Ollama 本地选项（完全免费）。所有 API 调用在 Rust 端完成，Key 不进前端 JS。

| # | Task | 详细说明 |
|---|------|---------|
| 1 | **`ai.rs` 模块** | Rust 端 API Key 密钥链存取（`keyring` crate，Windows Credential Manager / macOS Keychain）；HTTP 客户端（`reqwest`，支持 OpenAI 兼容接口）；超时 + 自动重试 + Token 预估（`tiktoken-rs`） |
| 2 | **Prompt 模板 + 输出校验器** | 构造结构化 prompt（固定模板 + 玩家输入的设定文本）；接收 `.md` 返回后解析 frontmatter + 表格；校验必填字段、称号升序、事件 ID 唯一；格式错误时自动重试最多 2 次，带具体错误反馈 |
| 3 | **前端生成流程 UI** | Hub 中「AI 生成」入口按钮 → 弹出窗口：标题输入、设定文本框（支持粘贴）、支持拖入 `.txt` 文件（读文件内容作为设定）、语言选择（zh/en）、取消按钮；生成阶段式伪进度（收集信息→生成事件→校验→完成）；生成后调用 `get_scenario_list` 刷新副本列表 |
| 4 | **副本导入导出** | 拖入 `.md` 文件 → 校验格式 → 加入列表；导出副本为 `.md` 文件；删除副本时清理关联存档数据 |
| 5 | **设置面板扩展** | API Key 输入（密码框，掩码显示）；API 地址配置（默认 DeepSeek）；模型选择；Ollama 本地开关；测试连接按钮 |

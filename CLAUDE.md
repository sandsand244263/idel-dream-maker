# CLAUDE.md — Idel-DreamMaker 项目主文档

**新 AI 会话请先读此文件。所有游戏规则、架构约定和开发约束在此定义。**

---

## 项目一句话定义

> Idel-DreamMaker = 藏在系统托盘里的挂机叙事陪伴应用。**大厅 (Hub) + 剧本内 (Scenario)** 双层架构：在大厅管理剧本，进入剧本后零交互挂机升级、随机弹出故事事件。

---

## 核心规则（AI 必须遵守）

1. **所有事件文本不在代码里硬编码**，由 AI 批量生成后放入剧本文件（`.md` 格式，详见`剧本MD格式模板.md`）
2. **游戏是零交互的**——进入剧本后没有按钮、点击、选择分支。纯挂机 + 偶尔阅读
3. **Steamworks 集成为可选项**，延后到 1.0，代码里条件编译
4. **跨平台**——Windows 系统托盘，macOS 菜单栏图标
5. **所有数据存为本地文件**（JSON 存档 + 本地剧本文件），不做云端数据库
6. **存档位置**：`%APPDATA%/Idel-DreamMaker/`（Win）或 `~/Library/Application Support/Idel-DreamMaker/`（Mac）

## 双层架构

### 大厅 (Hub)

大厅是整个应用的"家"。玩家在这里管理一切：

| 功能 | 说明 |
|------|------|
| 全局玩家名 | 首次创建，大厅中唯一标识 |
| 剧本列表 | 已拥有的剧本（卡片列表），可展开查看详情 |
| 剧本选择 | 从列表中直接点击进入 |
| 剧本抽取 | 从所有可用剧本中随机选一个 |
| 导入剧本 | 导入 `.md` 格式剧本文件 |
| AI 生成剧本 | 自备 API Key，AI 按模板生成 `.md` 剧本 |
| 删除剧本 | 删除剧本时同步清理关联存档数据 |
| 大厅称号 | 各剧本已解锁称号的聚合视图，按剧本分组可展开/折叠 |
| 大厅等级 | 各剧本累计总经验决定的全局等级，永不重置 |

**大厅内规则：** 不涨 EXP、不触发事件、不检测成就。纯管理界面。

### 剧本内 (Scenario)

进入剧本后，进入纯挂机模式：

| 功能 | 说明 |
|------|------|
| 剧本别名 | 每次进入可自定义名称（独立于全局名），仅该剧本内使用 |
| 等级 | 剧本内独立等级，退出/切换时重置 |
| EXP | 1/s 恒定，仅在本剧本内累积 |
| 称号 | 剧本专属称号链，等级达标自动解锁 |
| 事件 | 随机触发剧本专属故事事件 |
| 成就 | 剧本专属成就 |
| 返回大厅 | 随时可退出，退出时剧本等级重置，**总经验累加入大厅总经验** |

**剧本内规则：** 零交互挂机，只读。

### 大厅与剧本切换流程

```
大厅（Hub）
├── 选择剧本 → 进入（可选别名）
│   └── 剧本内：挂机积累 EXP/等级/称号/事件/成就
│       └── 退出 → 等级重置，该剧本获得的 EXP 加入大厅总经验
│           └── 大厅等级重新计算
│
├── 抽取剧本 → 随机给一个（免费，不限次数）
│   └── 同选择流程
│
├── 导入剧本 → 拖入 .md 文件 → 验证格式 → 加入列表
├── AI 生成 → 输入小说/设定 → AI 生成 .md → 加入列表
└── 删除剧本 → 从列表中移除 + 清理该剧本存档数据
```

## 数据规格

| 项目 | 数值 | 说明 |
|------|------|------|
| EXP 速率 | 1 EXP/s | 恒定不变，仅剧本内生效 |
| 大厅等级公式 | `floor(sqrt(hub_total_exp / 100)) + 1` | 全局进度，永不重置 |
| 剧本等级公式 | `floor(sqrt(剧本内总经验 / 100)) + 1` | 退出时重置，经验累入大厅 |
| 称号等级段 | 30 级/剧本 | 约 1-2 天解锁一级 |
| 事件文本 | 500 条/剧本 | AI 批量生成 |
| 成就 | 50 个/剧本 | 等级/时长/事件/收集四类 |
| 日志保留 | 不限 | 内存中保留全部，界面滚动查看 |
| 存档 | 多存档 | 大厅存档（全局）+ 各剧本存档 |

**大厅称号：** 各剧本已解锁称号的聚合视图，按剧本分组，UI 可展开/折叠。

## 大厅等级

```
每次退出剧本时：
  hub.total_exp += scenario.exp_earned_this_run
  hub.level = calculate_level(hub.total_exp)

其中：
  calculate_level(exp) = floor(sqrt(exp / 100)) + 1
```

- 大厅自身挂机不涨经验
- 切换剧本时同步累加
- 大厅等级**永不重置**

## 剧本 `.md` 格式

每个剧本一个 `.md` 文件，通过 frontmatter（YAML）定义元数据 + 正文分节定义内容。

详见 `剧本MD格式模板.md`。核心结构：

```yaml
---
id: wasteland
name: Wasteland
name_cn: 废土
language: zh
description: 核战后的废土世界...
player_title: 拾荒者
---
```

正文使用 Markdown 标题分割各区块（`## Titles`、`## Events`、`## Achievements`）。

## 事件触发逻辑

```
每 60 秒检查一次
触发概率：前 12 小时 40%/次，之后 30%/次，72 小时后 15%/次
平均间隔：前期 ~2.5 分钟，中期 ~3.3 分钟，后期 ~6.7 分钟

选择事件：
  1. 从当前剧本的事件池抽取
  2. 过滤：minLevel <= 当前等级，minHours <= 已运行小时数
  3. once=true 的事件只触发一次
  4. 按 weight 权重随机选择

弹窗行为：
  右下角弹出，持续 6 秒自动消失
  成就弹窗更突出，持续 8 秒
  可点击弹窗提前关闭
  每个事件同时触发系统通知（托盘气泡）
```

## 语言系统

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `language` | UI 界面语言（按钮/标签/提示等） | `zh` |
| `ai_output_language` | AI 生成剧本时的输出语言 | `zh` |

当前支持：`zh`（简体中文）、`en`（English）。

- 支持选择输出语言

# ScenarioWriter 副本格式规则

本文件定义 .md 副本文件的核心格式规范，同时被 SKILL.md 和验证脚本引用。

## Frontmatter 字段

```yaml
---
id: wasteland           # 必填。小写字母/数字/下划线
name: Wasteland         # 必填。英文名
name_cn: 废土           # 必填。中文名
description: 核战后的废土世界...  # 必填。世界观描述
player_title: 拾荒者    # 必填。Lv.1 默认称号
language: zh            # 必填。zh
mechanic: standard      # 可选。挂机变体：standard/cultivation/cyber/tide/polar，默认 standard
max_rebirth: 3          # 可选。最大重生次数 0-3，默认 0
unlock_requirement:      # 可选。解锁条件，默认全 0（默认解锁）
  hub_level: 0          # 大厅等级要求
  completions: 0        # 任意副本通关次数要求
completion_title: 废土行者  # 可选。通关解锁的大厅称号
---
```

## Titles 表规则

| 列 | 必填 | 类型 | 说明 |
|----|------|------|------|
| Level | 是 | 整数 | 解锁等级，必须升序 |
| Name | 是 | 字符串 | 称号名称 |
| Color | 是 | 字符串 | 十六进制含 `#` |
| Description | 是 | 字符串 | 称号描述 |

要求：**30 个**，Level 严格升序。称号跨周目共用，不区分周目。

## Events 表规则

| 列 | 必填 | 类型 | 说明 |
|----|------|------|------|
| ID | 是 | 字符串 | `{id}_e{四位编号}`，choice target 用 `e{编号}a` / `e{编号}b` |
| Type | 是 | 枚举 | `story`（剧情节点，升级时触发）或 `filler`（日常填充，时间驱动） |
| MinLevel | 否，默认 1 | 整数 | 对应等级编号 1-500 |
| MinHours | 否，默认 0 | 整数 | 固定为 0 |
| Weight | 否，默认 1 | 整数 | 固定为 1 |
| Once | 否，默认 是 | 是/否 | 全部为 是 |
| MinRebirth | 否，默认 0 | 整数 | 专属周目（0=首周，1=重生1次后，2=重生2次后，3=重生3次后） |
| Choice1 | 否 | 字符串 | 选项 A 的按钮文字（10 字以内），有值则本条为 choice 事件 |
| Choice1Target | 否 | 字符串 | 选 A 后触发的事件 ID，必须存在 |
| Choice2 | 否 | 字符串 | 选项 B 的按钮文字（10 字以内） |
| Choice2Target | 否 | 字符串 | 选 B 后触发的事件 ID，必须存在 |
| Action | 否 | 字符串 | 核心动作关键词（如"翻找"、"修水管"），用于校验文本多样性 |
| Text | 是 | 字符串 | 30-80 字 |

### 选择事件（choice）

choice 事件 = 在 story 事件上加 Choice1/Choice1Target/Choice2/Choice2Target 列，让玩家在关键节点做选择。

**规则：**
- Choice1 有值 → 本条视为 choice 事件，bubble 显示两个按钮
- Choice1Target 指向目标事件 ID，目标事件必须是同一副本内的 story 事件（不加 Choice 列）
- Choice2Target 同理，可省略（单选项）
- 目标事件生成时用 `{编号}a` / `{编号}b` 作为 ID，插入在 choice 事件等级之后
- 推荐每周目 20-25 个 choice，均匀分布在 500 级中

**两种 choice 类型：**

| 类型 | 影响范围 | 数量/周目 | 说明 |
|:----|:--------|:---------:|------|
| **重大抉择** | 后续 15 级的故事走向 | 6 个 | 选 A 后 15 级走 A 路线故事，选 B 走 B 路线，15 级后汇合 |
| **快速选择** | 仅下一条事件 | 14-19 个 | 小插曲，不影响后续主线 |

**例：**
```
| ID | Type | MinLevel | ... | Choice1 | Choice1Target | Choice2 | Choice2Target | Text |
| w_e050 | story | 50 | ... | 走大道 | w_e051a | 穿密林 | w_e051b | 你站在岔路口... |
| w_e051a | story | 51 | ... | | | | | 你走上大道，遇到商队... |
| w_e051b | story | 51 | ... | | | | | 你钻入密林，踩到捕兽夹... |
```

### 事件类型

| 类型 | 数量 | 触发方式 | 内容风格 |
|:----|:----:|:---------|:---------|
| `story`（普通） | 460~475/周目 × (1 + max_rebirth) 周目 | 升级时触发，与等级提示合并 | 剧情主线，推动人生叙事 |
| `story`（choice） | 20-25/周目 × (1 + max_rebirth) 周目 | 升级时触发，带两个选项按钮 | 关键转折点，玩家选择决定后续走向 |
| `choice target` | 40-50/周目 × (1 + max_rebirth) 周目 | 由玩家选择后立即弹出 | 选择后的分支叙事 |
| `filler` | 1365/周目 × (1 + max_rebirth) 周目 | 每天首次登录 1 条 + 挂机每小时 1 条，上限动态：8 + floor(挂机小时/4) + 每日仪式2 + 重生加成 | 该周目下的日常见闻，不推进剧情 |

### filler 密度公式

公式：`filler 数量 = floor(升该级所需分钟数 / 30)`

上班族每天挂机 8 小时、每小时弹 1 条 filler，一天最多看到 8~12 条。单趟（周目0）需挂机 117 天，总计看到 ~1170 条 filler。公式按每 30 分钟配 1 条，全等级合计 1365 条，足以覆盖一趟完整人生。

### 周目与 filler 的对应关系

每个周目有自己专属的一套 filler（通过 MinRebirth 列区分），描述该周目生活方式的日常碎片。

| MinRebirth | 周目 | 说明 |
|:---------:|:----:|------|
| 0 | 周目0（主线） | 主线人生的日常 |
| 1 | 周目1（番外） | 另一种活法的日常 |
| 2 | 周目2（另一视角） | 再一种活法的日常 |
| 3 | 周目3（隐藏结局） | 终局的日常 |

每个周目独立生成 1365 条 filler，各周目间故事独立、日常独立。

### filler 生成规则（替代旧防重复规范）

filler 使用"模板池填空"方式生成，而非自由创作。

1. **模板池必须复用**：从 Stage 1.6 设计的 15 个模板中选择，逐条填空生成。同一模板连续使用不超过 3 次。

2. **变量值从锚点文件取**：每个填槽位的取值优先使用锚点文件中定义的`角色`、`地点`、`物件`。超出锚点的取值必须与锚点世界观一致。

3. **Action 列跟踪**：15 个模板各绑定一个唯一 Action 词（模板设计时保证不重复）。同一周目内 Action 不可出现连续 3 条相同。

4. **文本长度 30-80 字**：包括填空后的完整句子。

5. **每周目独立模板池**：4 周目共 60 个不同模板，各周目间风格自动区分。

### story 叙事节奏

| 阶段 | 等级 | story 数 | 叙事风格 |
|------|------|:-------:|---------|
| 开端 | 1-50 | 50 | 初始世界、初遇挑战 |
| 成长 | 51-150 | 100 | 结交盟友、确立目标 |
| 鼎盛 | 151-300 | 150 | 应对大敌、建功立业 |
| 沉淀 | 301-450 | 150 | 培养后辈、准备谢幕 |
| 终章 | 451-500 | 50 | 完成最后之事、坦然离去 |

此节奏为默认推荐，每个副本可在 Stage 1.5 调整各阶段等级范围和风格。各周目 story 均为 500 条、从头到尾逐级推进，只是故事内容不同。

### 重生番外结构

| 周目 | 名称 | story 数量 | filler 数量 | 合计 | MinRebirth | 说明 |
|------|------|:----------:|:----------:|:---:|:---------:|------|
| 周目0 | 主线 | 500 | 1365 | 1865 | 0 | 首次体验的完整故事 |
| 周目1 | 番外 | 500 | 1365 | 1865 | 1 | 重生1次后解锁，同一世界观下的另一种活法 |
| 周目2 | 另一视角 | 500 | 1365 | 1865 | 2 | 重生2次后解锁，又一种活法 |
| 周目3 | 隐藏结局 | 500 | 1365 | 1865 | 3 | 重生3次后解锁，终局视角 |

各周目 story 均为 500 条逐级分布（LV1→500 各一条），与主线结构一致。

### 结局文本

最后一条 story（LV500，MinRebirth=0）作为副本结局文本，在玩家达到 LV500 时自动弹出。应写成结局风格（坦然离去、旅程终点、总结回顾等）。

各周目的最后一条 story（LV500，对应 MinRebirth）也类似处理，但在当前游戏引擎中仅 MinRebirth=0 的结局有特殊弹窗。

## HolidayEvents 表规则

| 列 | 必填 | 类型 | 说明 |
|----|------|------|------|
| ID | 是 | 字符串 | `{id}_h{三位编号}` |
| HolidayID | 是 | 字符串 | 对应 `electron/holiday.cjs` 中的节日 ID |
| Type | 是 | 枚举 | `advance`（节前 3 天）或 `day`（节日当天） |
| MinLevel | 否，默认 1 | 整数 | 最低触发等级 |
| MinHours | 否，默认 0 | 整数 | 最低挂机小时数 |
| Weight | 否，默认 1 | 整数 | 固定为 1 |
| Once | 否，默认 是 | 是/否 | 固定为 是 |
| Text | 是 | 字符串 | 30-80 字 |

每个节日需要 2 条事件（1 条 advance + 1 条 day），共 26 个节日 52 条。
节日事件跨周目共用（MinRebirth=0）。

节日列表（26 个）：
- 固定日期（14）：new_year, valentine, women_day, white_day, april_fools, earth_day, labor_day, childrens_day, environment_day, peace_day, halloween, christmas_eve, christmas, new_year_eve
- 农历节日（7）：spring_festival, lantern_festival, dragon_boat, qixi, zhongyuan, mid_autumn, double_ninth
- 浮动节日（5）：qingming（清明节气）, easter（复活节）, mothers_day, fathers_day, thanksgiving

## Achievements 表规则

| 列 | 必填 | 类型 | 说明 |
|----|------|------|------|
| ID | 是 | 字符串 | `{id}_a{三位编号}` |
| Name | 是 | 字符串 | 成就名称 |
| Description | 是 | 字符串 | 成就描述 |
| Icon | 否，默认 ★ | 字符串 | 单字符图标 |
| ConditionType | 是 | 枚举 | level/runtime/events/titles |
| ConditionValue | 是 | 数值 | 见下方"条件值与单位" |
| MinRebirth | 否，默认 0 | 整数 | 所属周目（0=首周目，1=重生1次后，2=重生2次后，3=重生3次后）。均匀分配到各周目，让每个周目都有新成就可解锁 |

要求：**50 个**，四种 ConditionType 严格按以下数量分布：
- level: **13** 个
- runtime: **12** 个（值必须为毫秒）
- events: **12** 个
- titles: **13** 个

### 条件值与单位

| ConditionType | ConditionValue | 说明 |
|:---|:---|:---|
| `level` | 等级编号，如 5、50、200 | 达到指定等级时解锁 |
| `runtime` | **毫秒值** | 示例：`3600000`=1小时，`86400000`=24小时。最大值以项目 `CLAUDE.md` 数据规格章节的里程碑最高值为参考 |
| `events` | 事件数，如 10、50 | 触发过 N 个不同事件时解锁 |
| `titles` | 称号数，如 5、15 | 解锁 N 个称号时解锁 |

## 常见校验失败

- 事件 ID 重复（ID 检查）
- Title Level 未升序（升序检查）
- 事件文本 < 10 字（长度检查）
- weight 不在 1-10 范围内（范围检查）
- Frontmatter 缺少必填字段（字段检查）
- HolidayEvents 缺少 Type 列
- Events 缺少 Type 列
- Events Type 值不是 story 或 filler
- runtime 类型的 ConditionValue 不是毫秒值
